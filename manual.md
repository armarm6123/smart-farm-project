##  1. ขั้นตอนการติดตั้งและตั้งค่า MQTT Broker (Mosquitto) บน Linux

ระบบนี้ใช้โปรโตคอล MQTT ในการรับส่งค่าเซนเซอร์และสั่งการบอร์ดแบบเรียลไทม์ โดยจะใช้คอมพิวเตอร์ระบบปฏิบัติการ Linux ทำหน้าที่เป็นตัวกลาง (Broker) มีขั้นตอนการติดตั้งอย่างละเอียดดังนี้:

1. **อัปเดตดัชนีแพ็กเกจของระบบ:** เปิดหน้าต่าง Terminal ขึ้นมา แล้วพิมพ์คำสั่งเพื่ออัปเดตรายชื่อแพ็กเกจให้เป็นปัจจุบันที่สุดก่อนเริ่มติดตั้ง
   sudo apt update
2. **ติดตั้งบริการ Mosquitto:** พิมพ์คำสั่งเพื่อดาวน์โหลดและติดตั้งตัวบริการ MQTT Broker (Mosquitto) รวมถึงเครื่องมือสำหรับทดสอบการรับส่งข้อมูลผ่านคำสั่ง (mosquitto-clients)
   sudo apt install mosquitto mosquitto-clients -y
3. **ตั้งค่าให้เปิดใช้งานอัตโนมัติเมื่อเปิดเครื่อง:** ใช้คำสั่งเริ่มระบบเพื่อกำหนดให้ตัวบริการ Mosquitto เริ่มทำงานเองโดยอัตโนมัติทุกครั้งที่มีการเปิดเครื่องคอมพิวเตอร์ใหม่
   sudo systemctl enable mosquitto
4. **สั่งเปิดบริการในปัจจุบัน:** พิมพ์คำสั่งสั่งการให้ระบบเริ่มรันตัว MQTT Broker ให้พร้อมทำงานทันที ณ ตอนนี้
   sudo systemctl start mosquitto
5. **ตรวจสอบสถานะการทำงาน:** พิมพ์คำสั่งเช็คสถานะการทำงานของบริการ Mosquitto เพื่อให้มั่นใจว่าระบบรันอยู่ปกติโดยไม่มีข้อผิดพลาด (สังเกตสถานะสีเขียวต้องขึ้นคำว่า Active: active (running))
   sudo systemctl status mosquitto
6. **เปิดไฟล์คอนฟิกเพื่ออนุญาตการเข้าถึงจากภายนอก:** เปิดไฟล์ปรับแต่งคุณสมบัติหลัก of Mosquitto ด้วยโปรแกรมแก้ไขข้อความระดับระบบ
   sudo nano /etc/mosquitto/mosquitto.conf
7. **เพิ่มคำสั่งควบคุมพอร์ต TCP และพอร์ต WebSocket:** เลื่อนลงไปที่ท้ายสุดของไฟล์ แล้วทำการพิมพ์เพิ่มคำสั่งเพื่อเปิดพอร์ตสื่อสาร 2 รูปแบบ โดยเปิดพอร์ต 1883 สำหรับสัญญาณ TCP มาตรฐาน (ให้บอร์ดฮาร์ดแวร์และสคริปต์ Python วิ่งเข้ามาเชื่อมต่อ) และเปิดพอร์ต 9001 ในรูปแบบ WebSocket (เพื่อให้แอปพลิเคชันมือถือหรือหน้าเว็บสามารถสื่อสารรับส่งข้อมูลเรียลไทม์ผ่านเว็บเบราว์เซอร์และเครือข่ายยุคใหม่ได้โดยตรง) พร้อมตั้งค่าอนุญาตการเข้าถึงแบบไม่ต้องใส่รหัสผ่าน (allow_anonymous)
   listener 1883 0.0.0.0
   protocol mqtt

   listener 9001 0.0.0.0
   protocol websockets

   allow_anonymous = true
   *หมายเหตุ: เมื่อพิมพ์เสร็จแล้ว ให้กดปุ่ม Ctrl + O ตามด้วย Enter เพื่อบันทึกไฟล์ จากนั้นกด Ctrl + X เพื่อออกจากหน้าต่างระบบแก้ไขข้อความ nano*
8. **รีสตาร์ทตัวบริการ:** ทำการพิมพ์คำสั่ง Restart เซอร์วิส Mosquitto 1 ครั้ง เพื่อให้ระบบดึงเอาไฟล์คอนฟิกใหม่ที่เพิ่งแก้ไขไปใช้งาน
   sudo systemctl restart mosquitto
9. **ตรวจสอบหมายเลข IP Address:** พิมพ์คำสั่งแสดงพิกัดเครือข่ายของเครื่องคอมพิวเตอร์ Linux เพื่อนำเอาพิกัดเลข IP Address นี้ไปใช้อ้างอิงในแอปพลิเคชันมือถือ บอร์ดฮาร์ดแวร์ และสคริปต์ Python
   ip a

##  2. ขั้นตอนการตั้งค่าระบบสะพานเชื่อมข้อมูลด้วย Python (MQTT Bridge)

ทำหน้าที่เป็นสะพานเชื่อมคอยดักฟัง (Subscribe) ข้อมูลที่ส่งมาจากฟาร์ม ตรวจสอบสิทธิ์ความสัมพันธ์ แล้วนำไปเขียนบันทึกเก็บลงสู่ฐานข้อมูล MySQL มีขั้นตอนการตั้งค่าดังนี้:

1. **ติดตั้งตัวจัดการแพ็กเกจ Python:** พิมพ์คำสั่งติดตั้ง pip เพื่อเตรียมระบบสำหรับการดาวน์โหลดเครื่องมือเสริมของ Python
   sudo apt install python3-pip -y
2. **ติดตั้งไลบรารีใช้งาน:** พิมพ์คำสั่งติดตั้งโมดูลสำหรับจัดการโปรโตคอล MQTT (paho-mqtt) และไลบรารีตัวเชื่อมต่อฐานข้อมูล MySQL (mysql-connector-python)
   pip3 install paho-mqtt mysql-connector-python
3. **สร้างไฟล์สคริปต์ทำงาน:** ทำการสร้างไฟล์ข้อความเพื่อเขียนโค้ดระบบขึ้นมาใหม่ตั้งชื่อว่า mqtt_bridge.py
   sudo nano mqtt_bridge.py
4. **วางซอร์สโค้ดจัดการระบบ:** นำซอร์สโค้ดภาษา Python ใส่ลงในไฟล์ โดยภายในตัวไฟล์จะมีการระบุคอนฟิกเชื่อมต่อฐานข้อมูล (smart_farm_db ซึ่งอยู่บนระบบ Linux), และระบุหมายเลขไอพีของตัวMQTT_BROKER = "172.20.10.4" ให้แก้ไขเป็นหมายเลข IP Address จริงของคอมพิวเตอร์โฮสต์ Linux (ที่ดูได้จากคำสั่ง `ip a`) 
5. **สร้างไฟล์ระบบเซอร์วิสเบื้องหลัง:** สร้างไฟล์บริการระบบของ Linux ขึ้นมาใหม่ เพื่อให้สคริปต์ทำงานเบื้องหลังโดยอัตโนมัติทันทีที่มีการเปิดเครื่องคอมพิวเตอร์
   sudo nano /etc/systemd/system/smartfarm-mqtt.service
6. **เพิ่มคุณสมบัติในไฟล์เซอร์วิส:** พิมพ์โครงสร้างคำสั่งภายในไฟล์ เพื่อกำหนดคำอธิบายระบบ, ชี้โฟลเดอร์ทำงานหลัก (WorkingDirectory) ไปยังตู้พิกัดไฟล์จริง, กำหนดคำสั่งสั่งรันไฟล์ (ExecStart), และระบุเงื่อนไขให้ตัวบริการพยายามรีสตาร์ทตัวเองเสมอเมื่อระบบหยุดเชื่อมต่อ
7. **รีโหลดทะเบียนบริการในระบบ:** สั่งให้ระบบปฏิบัติการทำการอัปเดตรายชื่อไฟล์เซอร์วิสตัวใหม่ที่เพิ่งเพิ่มเข้าไป
   sudo systemctl daemon-reload
8. **เปิดใช้งานระบบอัตโนมัติ:** สั่งลงทะเบียนให้ตัวเซอร์วิส smartfarm-mqtt.service เปิดการทำงานตัวเองอัตโนมัติตั้งแต่ตอนบูตเปิดเครื่องใหม่
   sudo systemctl enable smartfarm-mqtt.service
9. **สั่งรันระบบปัจจุบัน:** สั่งการให้สคริปต์ตัวสะพานเชื่อมข้อมูลเริ่มต้นทำงานมอนิเตอร์เครือข่ายดักค่าเซนเซอร์ทันที
   sudo systemctl start smartfarm-mqtt.service
10. **ตรวจสอบการทำงานของเซอร์วิส:** พิมพ์คำสั่งตรวจดูความเรียบร้อยของระบบ เพื่อล็อกสถานะการเปิดใช้งานเบื้องหลังให้ถูกต้อง (Active: active (running))
    sudo systemctl status smartfarm-mqtt.service

##  3. ขั้นตอนการเตรียมระบบฐานข้อมูล (MySQL Setup) บน Linux

เนื่องจากระบบฐานข้อมูลทำงานอยู่บนระบบปฏิบัติการ Linux (ผ่านระบบจัดการฐานข้อมูล เช่น MySQL Server หรือ LAMPP/XAMPP for Linux) มีขั้นตอนการเตรียมระบบและนำเข้าไฟล์ดังนี้:

1. **เปิดรันบริการฐานข้อมูล:** พิมพ์คำสั่งเปิดใช้งานระบบฐานข้อมูล MySQL บน Linux Server (หรือสั่งเปิดบริการรัน Apache และ MySQL ผ่านระบบควบคุม XAMPP Control Panel ของ Linux) เพื่อเปิดสแตนด์บายพอร์ตเชื่อมโยงข้อมูล
2. **เข้าสู่หน้าเครื่องมือจัดการ:** เปิดเว็บเบราว์เซอร์จากเครื่องโฮสต์แล้วเรียกไปยังเครื่องมือดูแลจัดการ phpMyAdmin ผ่านพิกัดพอร์ตภายในเครื่อง: `http://localhost/phpmyadmin/`
3. **สร้างฐานข้อมูลรองรับระบบ:** คลิกเลือกปุ่มคำสั่งสร้างฐานข้อมูลใหม่ (New) ที่แถบเมนูด้านซ้ายมือ โดยพิมพ์กำหนดชื่อระบุตรงตัวว่า smart_farm_db
4. **นำเข้าโครงสร้างระบบ:** คลิกเลือกที่ชื่อฐานข้อมูล smart_farm_db จากนั้นให้กดเลือกเปิดแท็บเมนู นำเข้า (Import) บริเวณด้านบน
5. **โหลดข้อมูล:** กดปุ่มค้นหาเลือกไฟล์ (Choose File) แล้วเลือกไฟล์ข้อมูลชื่อ smart_farm_db.sql ที่บรรจุอยู่ภายในตัวโปรเจกต์
6. **ประมวลผลคำสั่งสำเร็จ:** เลื่อนหน้าจอลงมาด้านล่างสุดแล้วกดคลิกปุ่ม นำเข้า (Import) ระบบฐานข้อมูลบน Linux จะสั่งรันคำสั่ง SQL

##  4. ขั้นตอนการติดตั้ง Web Dashboard และ API หลังบ้าน (PHP Deployment)

1. ไปยัง (`/opt/lampp/htdocs`) และสร้างโฟลเดอร์ใหม่ชื่อ project 
2. คัดลอกโฟลเดอร์ api ที่อยู่ในโฟลเดอร์ 2_backend_api ไปจัดวางไว้ภายในไดเรกทอรีโปรเจกต์ เพื่อให้ระบบจับคู่พิกัดโฟลเดอร์ปลายทางได้ถูกต้อง (ใส่ใน`/opt/lampp/htdocs/project/`)
3. สำหรับหน้าจอเว็บแดชบอร์ด ให้ทำการจัดส่งไฟล์สคริปต์ PHP ทั้งหมดในโฟลเดอร์ 3_web_dashboard ใส่ไว้ในโฟลเดอร์ (`/opt/lampp/htdocs/project`)
4. เปิดตรวจสอบไฟล์เชื่อมต่อฐานข้อมูล(db_config.php) เพื่อเช็คให้มั่นใจว่าตัวแปรระบุพิกัดชี้ตรงไปยังตัวดาต้าเบส MySQL บน Linux เรียบร้อยแล้ว (ชื่อผู้ใช้เป็น root, รหัสผ่านถูกต้อง และชื่อฐานข้อมูลระบุตรงกับมราสร้าง)
5. ทดสอบเรียกเปิดหน้าจอล็อกอินผ่านเว็บเบราว์เซอร์ที่ URL พิกัดเว็บปลายทาง เช่น `http://172.20.10.4/project/web_login.php` ให้เปลี่ยนหมายเลข IP Address เป็นของคอมพิวเตอร์โฮสต์ Linux (ที่ดูได้จากคำสั่ง `ip a`) เพื่อทดลองใช้งาน

##  5. ขั้นตอนการตั้งค่าแอปพลิเคชันมือถือ (Flutter Client Configuration)

1. **ปรับแต่งพิกัด API หลักและโครงสร้างโฟลเดอร์โปรเจกต์:** เปิดโปรเจกต์เข้าสู่ไฟล์ `lib/config.dart` โดยทำการเปลี่ยนจากคำว่า `localhost` ให้เป็นหมายเลข IP Address จริงของคอมพิวเตอร์โฮสต์ Linux (ที่ดูได้จากคำสั่ง `ip a`) และคงส่วนพ่วงท้ายไดเรกทอรีย่อยตามสถาปัตยกรรมโครงสร้างโปรเจกต์จริง ตัวอย่างเช่น เปลี่ยนจาก `http://localhost/project/api` เป็น `http://172.20.10.4/project/api`
2. **ปรับแต่งพิกัดเครือข่ายเรียลไทม์ผ่าน** เปิดเข้าสู่ไฟล์ `lib/mqtt_manager.dart` มองหาจุดประกาศค่าตัวแปรเครือข่าย เช่น final String host = "172.20.10.4"; ให้เป็นหมายเลข IP Address จริงของคอมพิวเตอร์โฮสต์ Linux (ที่ดูได้จากคำสั่ง `ip a`) 
3. เปิดหน้าต่าง Terminal ของเครื่องมือพัฒนาโปรเจกต์ Flutter แล้วพิมพ์คำสั่งเพื่อดาวน์โหลดไฟล์แพ็กเกจส่วนประกอบที่ระบบจำเป็นต้องใช้งาน
   flutter pub get
4. ตรวจสอบให้แน่ใจว่าอุปกรณ์สมาร์ตโฟนและเครื่องคอมพิวเตอร์เซิร์ฟเวอร์ Linux ต่อเครือข่าย Wi-Fi วงแลนตัวเดียวกัน จากนั้นใช้คำสั่งสั่งคอมไพล์เพื่อรันตัวแอปพลิเคชันขึ้นมาทดสอบระบบ
   flutter run

##  6. ขั้นตอนการตั้งค่าเว็บ (Web Configuration)

1. **ปรับแต่งพิกัด API หลักและโครงสร้างโฟลเดอร์โปรเจกต์:** เปิดสคริปต์ไฟล์ชื่อ user_farm.php (ที่จัดวางไว้ในโฟลเดอร์ /opt/lampp/htdocs/project/ บนเครื่อง Linux)
2. **ปรับแต่งพิกัดเครือข่ายเรียลไทม์ผ่าน WebSocket:** ทำการปรับเปลี่ยนหมายเลข IP Address ภายในตัวแปร brokerUrl ให้ชี้พิกัดไอพีตรงไปยังเครื่องคอมพิวเตอร์ Linux พร้อมระบุหมายเลขพอร์ตไปที่พอร์ต 9001 ซึ่งเป็นรูปแบบโปรโตคอล WebSocket (ws://) สำหรับใช้งานบนเว็บเบราว์เซอร์ แก้ไขจากเดิมให้เป็นหมายเลข IP Address จริงของคอมพิวเตอร์โฮสต์ Linux (ที่ดูได้จากคำสั่ง `ip a`) เช่น const brokerUrl = 'ws://172.20.10.4:9001'; เพื่อให้ระบบหน้าจอเว็บแดชบอร์ดสามารถเกาะสัญญาณดักฟังค่าเซนเซอร์และสั่งการเปิด-ปิดอุปกรณ์ร่วมกับตัว Broker ได้อย่างสมบูรณ์และเรียลไทม์

## 7. ขั้นตอนการตั้งค่าและการอัปโหลดโค้ดสำหรับบอร์ด ESP32 (ESP32 Firmware Configuration)

1. เปิดไฟล์ซอร์สโค้ดของโปรเจกต์ ที่อยู่ในโฟลเดอร์ `5_hardware_esp32` (ชื่อไฟล์ `ESP32.ino`) ในโปรแกรม **Arduino IDE**
2. **การตั้งค่าเครือข่าย Wi-Fi:**
   * แก้ไขตัวแปร `ssid` ให้เป็นชื่อเครือข่าย Wi-Fi ที่ต้องการใช้งาน (const char* ssid = "`your WIFI Name`";)
   * แก้ไขตัวแปร `password` ให้เป็นรหัสผ่านของเครือข่ายนั้น (const char* password = "`your WiFi Password`";)
   * ตรวจสอบให้มั่นใจว่าเป็นการเชื่อมต่อเข้าวงแลน (LAN) เดียวกับเครื่องคอมพิวเตอร์ Linux ที่ติดตั้ง Mosquitto Broker ไว้
3. **การตั้งค่ารหัสระบุตัวตน (Device Credentials):**
   * ค้นหาตัวแปรในโค้ดและแก้ไขค่าให้ตรงกับข้อมูลที่ได้จาก App/Web:
     * กรอก **UserToken** ของผู้ใช้งานลงในตัวแปร: (const char* userToken = "`your TokenID`";)
     * กรอก **SerialNumber** ที่ได้จากการสร้างฟาร์มลงในตัวแปร: (const char* serialNumber = "`your Serial Number`";)
   * การตั้งค่านี้สำคัญมาก เพื่อให้ระบบหลังบ้านระบุตัวตนฟาร์มของคุณได้อย่างถูกต้อง
4. **การตั้งค่า MQTT Broker:**
   * แก้ไขตัวแปร `mqtt_server` ให้ระบุหมายเลข **IP Address** ของเครื่องคอมพิวเตอร์ Linux ที่ติดตั้ง MQTT Broker เอาไว้ (const char* mqtt_server = "`your mqtt IP`";) เพื่อให้บอร์ดส่งข้อมูลเซนเซอร์ไปยังตัวกลางได้อย่างถูกต้อง
5. ตรวจสอบพอร์ตการเชื่อมต่อในคำสั่ง `client.setServer(mqtt_server, 1883);` ให้เป็นพอร์ต **1883** ซึ่งเป็นพอร์ตโปรโตคอล MQTT มาตรฐาน
6. ตรวจสอบความถูกต้องของการเชื่อมต่อสายไฟให้ตรงกับพินที่ประกาศไว้ในโค้ด (เช่น DHTPIN 32, SOIL_PIN 33, RELAY_LIGHT 18, RELAY_PUMP 19 และพิน I2C 21, 22 สำหรับจอ OLED)
7. เชื่อมต่อบอร์ด ESP32 เข้ากับคอมพิวเตอร์ผ่านสาย USB เลือกบอร์ดให้ตรงรุ่น (เช่น **ESP32 Dev Module**) และเลือกพอร์ตเชื่อมต่อให้ถูกต้องก่อนกดปุ่ม **Upload** เพื่อติดตั้งเฟิร์มแวร์ลงสู่บอร์ด
8. เมื่ออัปโหลดสำเร็จ ให้สังเกตหน้าจอ OLED ที่เชื่อมต่อกับบอร์ด หากตั้งค่าถูกต้องหน้าจอจะแสดงสถานะ WiFi เป็น "OK" และแสดงค่าจากเซนเซอร์แบบเรียลไทม์พร้อมสถานะการทำงานของอุปกรณ์ในฟาร์ม
